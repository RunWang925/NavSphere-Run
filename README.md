

# NavSphere 导航站 - 搜索体验深度重构版

本项目基于 [NavSphere](https://www.google.com/search?q=https://github.com/NavSphere/NavSphere) 原版进行重构。主要改进在于**彻底摒弃了 CMD+K 风格下拉弹窗搜索，改为基于页面原位的排他性卡片展示模式**，并深度优化了双模搜索逻辑与 UI 细节。


-----

## 🚀 视觉与交互重构 (UI/UX Re-architecture)

这是本次修改最核心的区别：

| 比较项 | 原版搜索 (Old) | **重构后搜索 (New)** |
| :--- | :--- | :--- |
| **显示形式** | **下拉浮窗 (Popover/Modal)** <br>遮挡页面内容，承载信息量有限。 | **原位排他展示 (In-place Exclusive)** <br>直接利用页面主体区域展示结果，视觉更开阔。 |
| **列表状态** | 输入文字时，背景列表依然可见但被遮挡。 | 输入文字时，**原有分类列表临时完全销毁**，仅展示匹配到的卡片。 |
| **视觉一致性** | 搜索结果样式与普通卡片不同（列表式）。 | 搜索结果复用原有的**标准卡片样式**，高亮显示关键词，视觉体验高度统一。 |
| **重置逻辑** | 关闭弹窗。 | 清空文字或点击 **X**，结果区域销毁，原分类列表**无缝瞬间回显**。 |

-----

## ✨ 功能改进 (Functional Improvements)

1.  **双模搜索切换**：
      * **Enter**：默认触发外部百度搜索。
      * **Ctrl + Enter**（或点击图标）：触发原位站内卡片搜索。
2.  **智能 UI 细节**：
      * **主题同步**：提示框背景色随系统深/浅模式实时自动切换。
      * **极致精简**：移除冗余的物理按键块，采用磨砂玻璃风格的悬浮提示引导。

-----

## 📂 项目结构 (Key Files)

  * **`components/search-bar.tsx`**
      * *功能*：搜索框 UI、按键监听、清除逻辑、**点击图标触发站内搜索**、纯 CSS 提示框实现。
  * **`components/navigation-content.tsx`**
      * *功能*：搜索状态中枢，**控制搜索结果卡片与原分类列表的“排他性”交替渲染**。
